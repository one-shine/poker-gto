# デザイン刷新計画 — 「Felt & Brass(分析ターミナル)」

> 親計画: [../IMPLEMENTATION_PLAN.md](../IMPLEMENTATION_PLAN.md)
> Phase 3 完了後の UI ポリッシュ。`frontend-design` スキルで方向付け。
> 純粋 Tailwind CSS 4 + Framer Motion 12 で実装(shadcn は Phase 4 で導入)。

## コンセプト

カジノの派手さではなく **プレミアムなポーカー分析端末**。落ち着いた深いダーク
+ フェルトグリーン + 真鍮(ブラス)アクセント。データ(BB/%/EV)は等幅フォントで
「端末」感を出す。記憶に残る一点は **照明付きフェルトテーブル + ブラスの質感**。

- トーン: 上質 / 分析的 / 集中。減点法ではなく意図のある洗練。
- 1番の差別化: 照明付きフェルト + ブラス + 等幅データ表示。

## 厳守する制約

- **テスト64件を壊さない**: `role="img"` / `aria-label`(「A スペード」「コール 1BB (c)」
  「3BB (3BB)」「ベット額スライダー」等)/ 表示テキスト(「手番中」「レイズ 2.5BB」
  「オールイン」)をすべて維持。テストは構造ではなくこれらの契約に依存。
- **色のみ非依存**(CLAUDE.md ルール5): スート記号・形状・アイコンを色と必ず併用。
- **モバイル対応**: rem/%/vw、タップターゲット 44px 維持。
- `src/engine/` は触らない(React 非依存維持)。
- 純粋 Tailwind(shadcn は Phase 4)。Framer Motion は描画後の aria/テキストを変えない範囲で使用。

## 微調整(第2弾) — 実機確認で判明した4点

刷新後、ユーザーが実機で気づいた点を解消。

1. **カードを本物のトランプの絵に** — `CardDisplay.tsx` をフル・ピップ配置へ。`PIP_LAYOUT`(ランクごとの標準座標、下半分は180°回転)で数札2–10を標準配置、A=中央大スート、絵札J/Q/K=飾り文字+ミラースート。コーナー索引も左上+右下(180°)の双方向に。ピップ可読化のためサイズ拡大(sm/md/lg)。
2. **自分の手札が不明 / BTNが常に手前** — 調査の結果ボタンは正常回転(`DealerAgent.ts:194`)。真因は**ヒーローが右下固定で、最も目立つ下中央が対戦相手**だったこと。`PokerTable.tsx` の `SEAT_POS` でヒーロー(seat0)を**下中央**へ、残り5席を反時計回りに再配置。`PlayerSeat.tsx` にヒーロー「あなた」リボン+手札を `lg` に拡大。ボードカードも `md` に。エンジンは無変更。
3. **レンジ表が小さい** — `RangeGrid.tsx` をレスポンシブ大型化(`minmax(0,1fr)` の aspect-square セル、max-w-44rem)、文字 7px→11px、ブラスのランクヘッダ。色のみ非依存のため**各セル角に R/C/M トークン併記**(ルール5 + オンボーディングの約束を実装)、凡例も整合。
4. **分析・理論タブのアイコンがダサい** — `AppShell.tsx` の `NAV_ITEMS` を絵文字/グリフ→**lucide 風細線インラインSVG**に6タブ統一(currentColor/24 viewBox/aria-hidden)。`NavItem.icon: string` → `Icon: IconComponent` に変更(テスト非依存)。

全64テスト維持・build成功・Playwrightで4点とも実機確認済み。

### 追修正: カードの視認性(ピップ潰れ)

第2弾のフル・ピップ実装で **ピップとコーナー索引が重なり、7〜10は密集して潰れ**ていた。
一時 probe(`?probe=cards` で全ランクを sm/md/lg 一覧 → 3倍拡大で精査、確認後に撤去)で検証して修正:
- ピップを**カード端からインセットした専用フィールド**に配置(`inset-x-[26%] inset-y-[15-16%]`)し、索引と分離。
- ピップ glyph を px 指定で適正サイズ化、座標を 2列+中央の標準配置に整理。
- **sm(相手カード)は密ピップを廃止**し、中央1スート+コーナー索引の簡易表示に(40px 級での可読性優先)。md/lg のみフルピップ。
- 絵札 J/Q/K は中央の大飾り文字。A は中央大スート1個。
- カードを一回り拡大(sm/md/lg)。実機(probe一覧 + ゲーム内ショーダウン)で潰れ解消を確認。

### 最終方針: ピップ廃止・ランク+スートのクリーン表示

インセット後も「文字の下の柄」が視認性を落としていたため、**フル・ピップを完全に廃止**。
本物のトランプ志向はやめ、**他のポーカーアプリ同様に「大きなランク + スート記号」だけ**の
クリーンなレイアウトに最終決定(`CardDisplay.tsx`):
- 中央に rank(display フォント極太・sm=text-lg/md=text-3xl/lg=text-5xl)+ 直下に suit 記号。重なり構造を排除。
- 色 + スート記号併用は維持(色覚配慮)。裏面はブラス菱格子+「G」のまま。
- `PIP_LAYOUT` / コーナー索引ロジックは削除。probe(`?probe=cards`)で全ランク sm/md/lg を確認後に撤去。
- 全64テスト維持・build成功・実機確認済み。

### 追修正: 相手アクションの視認性

「相手のアクションがわかりにくい」→ 小さなピル表示のみで、**ベット額が felt 上に見えない**のが主因だった。
他のポーカーアプリ同様の表現に強化:
- **ベットチップ層**(`PokerTable.tsx`): 各プレイヤーの `currentBetBB > 0` を、席〜中央へ寄せた位置に
  チップ+額(`betPos()` で補間)で表示。ブラインド/コール/レイズの「誰がいくら」が一目で分かる。
  `SEAT_POS` を数値化して補間に使用。ショーダウン中は非表示。
- **アクションバッジ強化**(`PlayerSeat.tsx`): 枠線のみ→**塗り**(fold=グレー/check·call=スカイ/raise=エメラルド/
  allin=ブラス)+ アイコン併用、`key` 変化で **spring ポップ**。視認性と色覚配慮を両立。
- 全64テスト維持・build成功・実機(ベットチップ 0.5/1/2.5 + 「▲ レイズ」緑バッジ)で確認済み。

### 追修正: レイアウト重なり + 文字サイズ

- **ヒントとカードの重なり**: ヒーロー席(下中央)の大カードが下のアクション領域(ヒントボタン)と重なっていた。
  `SEAT_POS[0]` の top を 88→80% に上げ(他席も微調整)、`GamePage` のセクション間 gap を 4→6 に拡大。重なり解消。
- **ポット/ベット額の文字拡大**: ポットラベルを pill 化し text-xs→text-base に。ベットチップの額を text-[11px]→text-sm、
  チップ径も拡大。※ポットラベルは「ポット {n}BB」を**単一テキストノードのまま**拡大(分割すると `getByText` テストが壊れるため)。
- 全64テスト維持・build成功・実機(ヒント開閉ともカードと非重複)で確認済み。

全64テスト維持・build成功・Playwrightで4領域を実機確認済み。
