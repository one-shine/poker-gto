# 競合比較 — GTO Atlas の優位性・強み・弱み

> **役割**: 他のGTO/ポーカーアプリと比べた本アプリ「GTO Atlas」の立ち位置・強み・弱みの一覧。
> 製品ポジショニングと収益化方針(②)の判断材料。誇張を避け、本アプリの精度の天井・範囲の狭さも明記する。
> **調査方法**: 5カテゴリ並列 Web 調査ワークフロー(2026-06-14)+ 本アプリの実装プロファイルと突合。
> **注意**: 競合の数値・仕様は調査時点のもので価格・機能は変動しうる。確度の弱い記述には「(要確認)」を付す。
> 関連: 公開準備 [`./RELEASE.md`](./RELEASE.md) / 仕様 [`./SPEC.md`](./SPEC.md) / 残課題 [`./BACKLOG.md`](./BACKLOG.md) C節(収益化)。

## ① 一言ポジショニング

**「無料・アカウント不要・完全ローカルで動き、解の確からしさを正直にラベル表示する、日本語の GTO 学習トレーナー」。**

精度の絶対量・スポット網羅では商用クラウド(GTO Wizard / DTO)に及ばないが、**「無料 × プライバシー完結 × 正直な精度開示 × 統合された学習ループ」を一つに束ねている点が独自**。競合は強い軸を1〜2本持つが、この4本同時を満たすものは見当たらない。

## ② 強み(なぜ他にないか・対象競合を併記)

### 1. 完全無料・広告なし・課金なし
- **対象競合**: GTO Wizard($49/月〜・無料枠は厳しく制限)、DTO(Cash Pro $39.99/月)、PokerSnowie($35〜$230/年)、PeakGTO($79/月)、デスクトップソルバー(買い切り $75〜€800)。
- **なぜ他にないか**: 上記はほぼ全てサブスクか有償買い切り。無料枠を持つものはあるが、**「無料 かつ ポストフロップをソルバー解で採点する統合トレーナー」**で本アプリほどの学習ループを無料で備える例は少ない。

### 2. アカウント不要・完全ローカル・第三者送信ゼロ
- **対象競合**: GTO Wizard(アカウント必須・完全クラウド・履歴をサーバーにアップロード)、DTO/PokerSnowie(アカウント必須・利用データ収集)。
- **なぜ他にないか**: クラウド型は構造上アカウントとデータ送信が前提。ローカル完結はデスクトップソルバーと Equilab だが、それらは Windows 中心・有償・玄人向け。**「アカウント不要 × ブラウザ(スマホ可)× オフライン × トラッキングなし」**を同時に満たすのは明確な空白地帯。

### 3. 解の信頼度を source ラベルで常時明示する「正直開示」設計
- **対象競合**: GTO Wizard(ベンチ数値で精度を示すが断定は避ける)、PokerSnowie(自己対戦NN近似で収束指標なし)、DTO(開発者自身「PIO に reasonably close」と近似的表現)。
- **なぜ他にないか**: 競合は精度の主張はしても、**解ごとに `solver_precomputed / solver_live / approximate` を UI とコードの両方で常時提示し、近似スポットでは "GTO最適" と断定しない**設計を製品の絶対ルールに据えた例は事実上見当たらない。「今見ている解がどれだけ信用できるか」を都度判別できるのは教育的に独自の価値。

### 4. ブラウザPWAでPC/スマホ/iPhone横断・オフライン動作
- **対象競合**: PioSOLVER/GTO+/Equilab(Windows中心、Mac は仮想化要)、PokerSnowie(モバイルは更新が古く機能限定・Postflop は2019年最終(要確認))。
- **なぜ他にないか**: デスクトップソルバー陣営はモバイル不可・Windows偏重。GTO Wizard/DTO はクロスプラットフォームだがクラウド前提でオフライン非対応。**インストール不要のブラウザPWAで、スマホでもオフラインでソルバー採点ドリルが回る**点は手軽さで効く。

### 5. ライセンス上クリーンな自社生成解(公開・商用可)
- **対象競合**: 事前計算配信型(GTO Wizard/DTO/PeakGTO)は自社ソルバー出力を持つが、第三者による再配布は不可。
- **なぜ他にないか**: 本アプリは**他社ソルバー出力を非同梱で全解を自前生成**。機能差ではなく、「無料公開・将来の商用化」を法的に取れる事業上の土台([`./DATA_LICENSE.md`](./DATA_LICENSE.md))。

### 6. 学習設計の統合度(ゲーム→分析→理論→ドリルの一周)
- **対象競合**: Equilab/Flopzilla(エクイティ計算のみ・戦略解なし)、デスクトップソルバー(求解特化で初心者導線が弱い)、PokerSnowie(分析は強いが理論講義は薄い)。
- **なぜ他にないか**: GTO Wizard と DTO は統合学習を高水準で実現しており**ここは差別化でなく同等を目指す土俵**。ただし本アプリは「手番前にこの局面の観点(位置/ボードテクスチャ/ハンドクラス/オッズ/サイズ/ブロッカー)を提示」「色覚配慮UI」など、**初心者の思考プロセスを言語化して支える日本語コーチング**として無料帯では厚い。

## ③ 弱み / 制約(正直に)

| 区分 | 本アプリの制約 | 競合がより強い点 |
|------|----------------|------------------|
| プリフロップ精度 | 100BB のオープン/3betは**真のナッシュではない近似(approximate)**。公開GTO RFIより約2〜3ptタイト。多人数の真Nashは原理的に不可と明記。 | GTO Wizard/DTOは事前計算の均衡解(固定パラメータ前提) |
| ポストフロップ精度の天井 | 厳密GTOは push/fold(≤25BB)と**代表フロップのみ**。ライブ求解は**単一ベットサイズ(2/3ポット)のみ**で、サイズ選択(レンジベットvsポラライズ)の厳密採点は未対応。 | デスクトップソルバーは任意ツリー・複数サイズをその場で厳密求解。GTO Wizardは多サイズ事前計算。 |
| スポット網羅 | 限定的(代表ボード中心)。巨大事前計算ライブラリは無い。 | GTO Wizard「数百万スポット/1,755フロップ」、DTO「1億スポット超」、PeakGTO「2000万超」 |
| ゲーム範囲 | 6-max NLHのみ。**マルチウェイはGTO精度から除外し参考値表示**。ICM非考慮・ノーレーク。PLO非対応。 | GTO WizardはPLO/ICM/PKO、Monkerはマルチウェイ/PLO、DTOは3-Way/MTT/ICM |
| カスタム求解 | ユーザーが任意ツリーを定義してその場で解く機能は無い(事前計算+限定ライブ求解)。 | デスクトップソルバー全般、GTO Wizard AI |
| ハンド履歴/HUD | **インポート・HUDなし**。実プレイのリーク分析はできない。 | PokerSnowie/GTO Wizard Analyzer/Simple Postflopはハンド履歴取込 |
| 対人・実マネー | オンライン対戦・実マネーなし。AIと対戦はあるが規模小。 | PokerSnowie/PokerArena(GTO Wizard)は対AI/対人 |
| 言語・規模 | **日本語のみ**。コミュニティ/学習グループ機能なし。個人開発・小規模(継続性・サポート面のリスク)。 | 商用各社は多言語・週次コーチング・運営体制あり |

**精度の天井(要点)**: 「厳密GTO」と呼べるのは push/fold と代表フロップのポストフロップ(exploitability中央値~0.02%=商用水準)に限られ、プリフロップ100BBと多サイズ・マルチウェイは近似または対象外。**広く浅い均衡ライブラリの代替にはならない**。

## ⑤ 勝てる土俵・負ける土俵・ニッチ

**勝てる土俵**
- **無料で始めたい日本語の初〜中級者**: 無料・アカウント不要・日本語・統合学習・色覚配慮。GTO Wizard無料枠(英語・登録必須・厳しい制限)より入口の摩擦が低い。
- **プレイデータを外に出したくない/オフラインで使いたい層**: クラウド型が原理的に取れず、ローカルソルバーは有償Windowsという空白に刺さる。
- **精度の限界を正直に学びたい層・指導者**: source ラベルで「どこまで信用できるか」を教えられる。
- **スマホでサクッと回したい人**: インストール不要PWAでオフライン採点ドリル。

**負ける土俵**
- 本格研究・任意スポット探索(デスクトップソルバー/GTO Wizard AI)。
- 広く浅くあらゆるスポット(GTO Wizard/DTO/PeakGTO の網羅)。
- MTT/ICM/PLO/マルチウェイの実戦最適化(DTO/Monker/GTO Wizard)。
- 実プレイのリーク診断(履歴/HUD)(PokerSnowie/GTO Wizard Analyzer)。
- 多言語・コミュニティ・運営の安心感(商用各社)。

**ニッチ(明確な居場所)**
> **「日本語話者の入門〜中級者が、お金もアカウントも要らず、データを端末から出さずに、プリフロップ基礎+ヘッズアップ・ポストフロップのGTO感覚を、解の確からしさを理解しながら掴むための学習トレーナー」。** 本格ソルバーへの "橋渡し・入口" として位置づけると競合と正面衝突しない。

## ⑥ 収益化方針への示唆(→ ② 収益化整理へ)

- **「無料・プライバシー・正直開示」は収益化と緊張関係**。これらは最大の差別化だが、サブスク化・広告・トラッキング導入は強みそのものを毀損する。
- 課金するなら、**精度・網羅では商用に勝てない以上「日本語の学習体験・コーチング・入門導線」に値付け**するのが筋。
- 無料を維持しつつ**寄付/買い切り(Pro機能)/有償の学習コンテンツ**など、データを外に出さないモデルと整合する手段に限定するのが安全。
- **自社生成解=ライセンスクリーンは商用化の前提条件として確保済み**。
- 競合と価格・網羅で殴り合わない。**「本格ソルバーへの入口・日本語・プライバシー完結」というニッチを収益の物語の中心に据える**のが、誇張せず守れる立て付け。
