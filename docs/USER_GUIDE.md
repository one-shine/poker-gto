# GTO Atlas 利用マニュアル

> ポーカー(6-max ノーリミットホールデム)の **GTO 戦略を学ぶための、完全ローカル動作の学習アプリ**です。
> 実マネーの賭け・換金・賞金はありません(教育・シミュレーション目的)。
> 製品仕様の詳細は [`SPEC.md`](SPEC.md) を参照。

---

## 1. これは何か

- AI 相手にハンドをプレイし、各局面で **GTO(ゲーム理論的最適)の推奨と「なぜ」をコーチが解説**します。
- レンジ表・ドリル・弱点分析・理論ライブラリで、実戦 → 復習 → 理論 → 練習の**学習ループ**を回せます。
- **解の信頼度を常に正直に表示**します(後述の `source` バッジ)。"絶対""GTO最適" のような断定はしません。
- データはすべて**端末内のみ**。アカウント不要・オフライン動作。

## 2. 起動方法

### ① 公開 URL で使う(最も簡単・推奨)
ブラウザで **<https://one-shine.github.io/poker-gto/>** を開くだけ。インストール・ビルド不要。

- **PC(Mac / Windows)**: **Chrome / Edge** で開き、アドレスバーの「インストール」→ 独立ウィンドウ + Dock/スタートアイコンで起動。
- **iPhone**: **Safari** で開き、共有メニュー →「ホーム画面に追加」。
- 初回ロード後は**オフラインでも起動**できます(PWA)。
- ※ このサイトは `noindex`(検索エンジンに出ない)です。**URL を知っている人だけ**がアクセスできます。

### ② iPhone(自分のローカルビルドを配信する場合)
公開 URL ではなく **自分の PC のビルド**を iPhone で使いたいときのフォールバックです。

- HTTPS トンネル(例 `cloudflared tunnel --url http://localhost:4173/poker-gto/`)で出た https URL を iPhone Safari で開く →「ホーム画面に追加」。初回ロード後はオフラインで起動可能。
- ※ iOS は Service Worker(オフライン)に HTTPS が必須です(公開 URL は HTTPS なのでこの条件を満たします)。

### ③ 開発者向け / 自分の PC で動かす
```bash
npm install        # 初回のみ
npm run dev        # 開発用:    http://localhost:5173/poker-gto/ (オフライン/PWA は無効)
# または本番同等(PWA・オフライン有効):
npm run build
npm run preview    # http://localhost:4173/poker-gto/
```
- ※ ベースパスが `/poker-gto/` のため、ルート(`/`)は 404 になります。上記の `/poker-gto/` 付き URL を開いてください。
- **アプリとして使う**: `npm run preview` を **Chrome / Edge** で開き、アドレスバーの「インストール」→ 独立ウィンドウ + Dock/スタートアイコンで起動。以後オフラインでも動作。

## 4. 基本の遊び方

1. **ゲーム**ページで **New Hand**(または `Space`)。
2. 自分の番になったら **フォールド / チェック・コール / ベット・レイズ** を選ぶ(プリセット・スライダー・キーボードも可)。
3. アクション後、**コーチ**が評価します。
   - **スタディモード**: ミス時にその場でパネルが開き、推奨頻度・EV 損失・「なぜ」「関連理論」を解説(一時停止)。
   - **プレイモード**: ハンドを止めず、重大ミスのみ通知。ハンド終了後に**ポストフロップ復習**(実ボードをその場で求解)。
4. 100 ハンドごとに**振り返り(リフレクション)**が自動表示。

### 2 つのモード軸(設定で切替)
- **フィードバックの濃さ**: スタディ(じっくり)/ プレイ(止めない)。
- **対戦相手**: トレーナー(vs GTO・乖離を測る)/ エクスプロイト(vs フィッシュ・実戦的)。

## 5. ゲーム画面の使い方

- **アクション操作**: ボタン + ベットサイズプリセット(プリフロップ=BB単位 / ポストフロップ=ポット% + 総額併記)+ スライダー。
- **GTO 戦略パネル**(スタディ): 各アクションの推奨頻度バー + EV。`設定`で常時表示/非表示を切替(非表示にすると実力測定として精度に算入)。
- **ポットオッズ / 必要勝率 / 自分のエクイティ**: 中級以上で表示(コール判断の学習補助)。
- **アクション履歴(ベットライン)**: そのハンドの「誰が何をしたか」をストリート別に表示。
- **コーチフィードバック**: EV 損失(BB)・ミス分類・概念ラベル・関連理論へのリンク。
- **ハンド後の復習**(プレイ): 実際のボードでポストフロップをソルバー求解して振り返り。

### キーボードショートカット
| キー | 操作 |
|------|------|
| `f` | フォールド |
| `c` | チェック / コール |
| `r` | ベット / レイズ |
| `Enter` | アクション確定 |
| `Space` | 次のハンドへ |
| `Escape` | パネルを閉じる |
| `?` | GTO 戦略パネルの表示切替(スタディ) |

## 6. 各機能

### 学習(ドリル)
- **プリフロップドリル**: 「このハンドはレイズ?フォールド?」を出題。頻度 10% 以上のアクションは正解扱い(ミックス戦略対応)。
- **ポストフロップドリル**: ランダムボードでソルバー解ベースの出題(実 EV 付き)。シングルレイズド/3betポット・ストリート切替。
- **プッシュ/フォールドドリル**: 5〜25BB の **7 段階**を厳密 Nash 解で出題(実 EV・厳密解判定)。
- 弱点カテゴリでフィルタ出題でき、理論記事の「ドリルで練習」から直接遷移できます。

### 分析
- **弱点 TOP3**: ミス記録から弱点カテゴリを抽出し、関連理論へ導線。
- **ポジション別 GTO 精度**: コーチが実評価したハンドのみで精度を算出(ヒント参照ハンドは除外、未評価は「データ不足」)。

### 理論
- 戦略**概念(約 25)** + **用語集(約 56)**。プリフロップ/ポストフロップ/数学/メンタルをカバー。盤面例・決定木・「よくあるミス」付き。

### レンジ
- **13×13 グリッド**: 混合戦略をセル内で頻度比に塗り分け(色のみ非依存・R/C/F トークン併記)。
- **レンジ比較 / エクイティ分布**: 2 レンジのボード上エクイティ分布・レンジ優位/ナッツ優位を可視化。

### 設定
- 対戦相手モード・フィードバックモード・スタック・自動再開秒数・GTO 戦略の常時表示・**サウンド/ハプティクス(既定 OFF)**・セッション振り返り。

## 7. 解の信頼度の読み方(重要)

各局面の解には**出典(`source`)バッジ**が付きます。本アプリの核心は「どこまでが厳密で、どこからが近似か」を隠さないことです。

| バッジ | 意味 | 主な対象 |
|--------|------|---------|
| **GTOソルバー解** | 同梱の**厳密解**(信頼度最高) | プッシュ/フォールド ≤25BB |
| GTOソルバー解(ローカル求解・簡易) | ブラウザ内 CFR で都度求解 | ポストフロップ(river=厳密 / turn=賭け考慮 / flop=簡易) |
| GTO理論準拠の近似 + 概算EV | GTO理論準拠の近似レンジ + 概算 EV(EV に `~`) | プリフロップ(オープン/ディフェンス/対3bet) |
| 参考: GTO近似 | GTO理論準拠の近似(EV なし) | 未取込スポット |

- **EV 損失(BB)** は数値で表示されます(ソルバー EV があるとき)。`参考: GTO近似` のときは数値を出さず文章解説のみ。
- 「flop は簡易(賭け未考慮)」など、精度が落ちる箇所は明示します。

## 8. 前提・制約

- **6-max・100BB 固定・ノーレーク・キャッシュ・ICM 非考慮**。トーナメント用途には不向き。
- **GTO 精度はヘッズアップ局面のみ**。3 人以上のマルチウェイは「参考値」と表示。
- 本物の厳密解はプッシュ/フォールド(≤25BB)。100BB のオープン/3bet は概算 EV 付き近似、ポストフロップはスタディ限定のローカル求解です(詳細 [`SPEC.md`](SPEC.md))。

## 9. データとプライバシー

- 学習データ(設定・XP・統計・ハンド履歴)は**端末内のみ**(localStorage / IndexedDB)。アカウント不要。
- 広告・トラッキングなし。フォントもセルフホストのため**第三者への通信ゼロ**(クラッシュレポートは既定 OFF)。
- 詳細は [`PRIVACY_POLICY.md`](PRIVACY_POLICY.md)。

## 10. アクセシビリティ

## 困ったとき
- 用語が分からない → **理論**タブの用語集、または各所の用語チップ(ⓘ)。
- 精度が「データ不足」 → コーチ評価対象のハンドが 20 未満のとき表示されます。プレイを続けると解消します。
- オフラインで動かない(iPhone)→ HTTPS 経由で一度開く必要があります。公開 URL(<https://one-shine.github.io/poker-gto/>)は HTTPS なので、Safari で一度開けば以後オフラインで起動できます(§2)。
